# Plan: 컬렉션 간 북마크 이동 드래그앤드롭

> Source PRD: [GitHub Issue #1](https://github.com/ihavefreeacount/My-Little-Bookmark/issues/1)

## Architectural decisions

Durable decisions that apply across all phases:

- **Routes**: 브라우저 `new tab` 컬렉션 보드 안에서만 동작한다. 새 라우트나 별도 화면은 추가하지 않는다.
- **Schema**: 영속 데이터 구조는 유지한다. 워크스페이스는 북마크 폴더, 컬렉션은 워크스페이스 하위 폴더, 북마크는 컬렉션 하위 URL 노드로 계속 표현한다.
- **Key models**: 워크스페이스별 컬렉션 요약, 컬렉션별 북마크 순서 맵, 북마크 드래그 세션, 드롭 타깃 프리뷰를 핵심 모델로 유지한다.
- **Persistence**: 이동 결과는 낙관적 UI 상태로 즉시 반영하되, 최종 정합성은 Chrome Bookmarks 트리 새로고침으로 맞춘다.
- **Third-party boundaries**: 북마크 이동과 동기화는 Chrome Bookmarks API 경계에서 처리하고, 포인터 기반 드래그 인터랙션은 현재 DnD 라이브러리 경계를 유지한다.
- **Scope boundary**: 이동 대상은 현재 선택된 워크스페이스에 렌더링된 컬렉션으로 제한한다. 워크스페이스 간 이동은 포함하지 않는다.

## Phase 1: 컬렉션 간 이동의 최소 경로

**User stories**: 9, 11, 12, 14, 15, 18, 19

### What to build

현재 워크스페이스 안에서 북마크를 한 컬렉션에서 다른 컬렉션으로 옮길 수 있는 최소 end-to-end 경로를 만든다. 이 단계에서는 “다른 컬렉션으로 이동이 실제로 저장되고 실패 시 복구된다”는 흐름을 우선 완성한다.

### Acceptance criteria

- [ ] 사용자는 현재 선택된 워크스페이스 안에서 북마크를 다른 컬렉션으로 드롭해 부모 컬렉션을 바꿀 수 있다.
- [ ] 이동 성공 시 보드가 새 상태로 동기화되고 이동 완료 토스트가 노출된다.
- [ ] 이동 실패 시 낙관적 상태가 실제 북마크 트리 기준으로 복구되고 실패 토스트가 노출된다.
- [ ] 자동화 테스트가 다른 컬렉션으로의 이동 호출과 성공/실패 복구 흐름을 검증한다.

## Phase 2: 채워진 컬렉션 안의 위치 지정

**User stories**: 1, 2, 3, 4, 7, 8, 16, 19

### What to build

북마크가 들어 있는 타깃 컬렉션에서도 기존과 동일한 좌/우 드롭 인디케이터를 보여주고, 특정 북마크 앞이나 뒤로 정확히 드롭할 수 있게 한다. 사용자는 컬렉션 경계를 넘어도 같은 정렬 감각으로 이동할 수 있어야 한다.

### Acceptance criteria

- [ ] 사용자가 북마크를 다른 populated collection 위로 드래그하면 해당 컬렉션에서도 좌/우 드롭 인디케이터가 표시된다.
- [ ] 첫 번째 북마크 앞과 마지막 북마크 뒤를 포함한 경계 위치가 정확한 대상 인덱스로 해석된다.
- [ ] 드롭 결과가 소스 컬렉션과 타깃 컬렉션 양쪽에 즉시 반영되어 사용자가 위치 변화를 확인할 수 있다.
- [ ] 자동화 테스트가 populated collection 간 위치 계산과 드롭 결과를 검증한다.

## Phase 3: 빈 컬렉션 드롭 존 완성

**User stories**: 5, 6, 8, 9, 11, 12, 20

### What to build

북마크가 없는 컬렉션도 드래그 대상이 되도록 확장한다. 사용자가 빈 컬렉션 위에 북마크를 올리면 카드 전체에서 드롭 가능 상태가 보여야 하고, 드롭 시 해당 컬렉션의 첫 번째 북마크로 저장되어야 한다.

### Acceptance criteria

- [ ] 빈 컬렉션 위로 드래그하면 카드 전체가 드롭 가능 상태로 표시된다.
- [ ] 빈 컬렉션에 드롭한 북마크는 대상 컬렉션의 첫 번째 항목으로 저장된다.
- [ ] 성공/실패 피드백과 복구 동작이 populated collection 이동과 동일한 수준으로 유지된다.
- [ ] 자동화 테스트가 빈 컬렉션 드롭 규칙과 index `0` 삽입을 검증한다.

## Phase 4: 회귀 방지와 UX 하드닝

**User stories**: 10, 13, 17, 19, 20

### What to build

이미 완성된 이동 흐름을 안정화한다. no-op 상황, 편집 중 입력 요소 예외, reduced motion 유지, 경계 케이스 회귀를 함께 정리해 사용자가 기능을 일관되게 신뢰할 수 있게 만든다.

### Acceptance criteria

- [ ] 원래 부모와 원래 시각적 위치에 다시 놓는 경우에는 저장 호출과 성공 토스트가 발생하지 않는다.
- [ ] 편집 중인 입력 요소나 드래그 예외 영역을 눌렀을 때는 이동 흐름이 시작되지 않는다.
- [ ] reduced motion 환경에서도 프리뷰와 드래그 피드백이 기존 접근성 기대를 깨지 않는다.
- [ ] 자동화 테스트가 no-op, 드래그 예외, 주요 경계 케이스 회귀를 보호한다.
